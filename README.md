# Лабиринт Теней: Память

## Дизайн-документ

### 1. Концепция

«Лабиринт Теней: Память» - двухмерная игра-лабиринт с элементами исследования. Игрок перемещается по случайно сгенерированному лабиринту, видит лишь небольшую освещённую область вокруг себя (фонарь). Пройденные клетки запоминаются и отображаются тусклым цветом, но память ограничена - старые воспоминания исчезают. В лабиринте обитают теневые стражи, которые преследуют игрока. Цель - найти выход из лабиринта, не попавшись врагам.

### 2. Жанр и платформа

- Жанр: 2D-лабиринт с элементами исследования
- Платформа: Windows
- Технологии: C#, Windows Forms, .NET 8

### 3. Основные механики

**Случайная генерация лабиринта**: каждый запуск создаёт новый лабиринт, гарантированно проходимый от старта до выхода.
**Ограниченная зона видимости**: игрок видит только небольшой квадрат вокруг себя (фонарь). Остальная часть лабиринта скрыта тьмой.
**Память клеток**: пройденные клетки запоминаются и отображаются затемнённой текстурой. Лимит памяти ограничен; при превышении лимита старые клетки забываются.
**Враги**: 5 теневых стражей.
**Главное меню и пауза**: возможность начать новую игру, выйти в главное меню, поставить игру на паузу (Esc).
**Таймер**: отсчёт времени с момента первого движения.

### 4. Враги

- **Патрульные (×4)**: блуждают по лабиринту случайным образом. Не привязаны к памяти игрока, появляются далеко от старта.
- **Преследователь (×1)**: появляется, когда игрок запомнил 7 клеток. Начинает движение от стартовой точки и идёт к игроку кратчайшим путём по запомненным клеткам. Быстрее патрульных.

### 5. Предметы

**Блестящий кристалл** +20 к лимиту памяти (6 сек) 
**Золотое зелье** +1 к радиусу обзора (3 сек) 
**Магические часы** Замедление всех врагов (3 сек) 

Предметы видны только в освещённой зоне. Всего генерируется по 2 предмета каждого типа.

### 6. Интерфейс

- **Игровое поле**: лабиринт с текстурами стен, пола, выхода, игрока, врагов, предметов.
- **Левая панель**: активные эффекты с оставшимся временем (вертикальный список с рамкой).
- **Правый верхний угол**: таймер в формате "MM:SS".
- **Финальный экран**: сообщение о победе/поражении, итоговое время, кнопки «Заново» и «Выход в главное меню».
- **Главное меню**: название игры, кнопки «Начать игру» и «Выход».
- **Меню паузы (Esc)**: полная видимость (отладка), новый лабиринт, выход в главное меню, закрыть.

### 7. Управление

- **W, A, S, D** или **Стрелки** - движение.
- **Esc** - пауза / меню настроек.

### 8. Структура проекта (MVC + Observer)

Проект разделён на три основные папки:

- **Model** - логика игры, независимая от интерфейса.
  - "Maze" - генерация и хранение лабиринта.
  - "Player" - позиция игрока, радиус обзора.
  - "MemorySystem" - очередь запомненных клеток с ограничением размера.
  - "Enemy" - поведение врагов (A*, патрулирование).
  - "Item", "ActiveEffect" - предметы и временные эффекты.
  - "GameModel" - главный класс модели, связывающий все компоненты.

- **View** - отрисовка и пользовательский интерфейс.
  - "MazeView" - вся графика: лабиринт, игрок, враги, предметы, меню, HUD.
  - "MainForm" - форма Windows Forms, обработка клавиш, запуск/пауза.

- **Controller** - посредник между вводом и моделью.
  - "GameController" - передача команд движения от клавиатуры в модель.

Модель общается с представлением через события (паттерн Observer): "PlayerMoved`, "MemoryChanged", "GameWon", "GameLost", "EnemiesUpdated", "ItemsUpdated", "EffectsUpdated". Представление подписывается на них и перерисовывает соответствующие элементы.

### 9. Используемые алгоритмы

- **Генерация лабиринта**: рекурсивное деление (Recursive Division) с дополнительной проверкой связности (BFS) и удалением изолированных областей и мусорных стен.
- **Поиск пути врагов**: A* по запомненным клеткам; если враг забыт - BFS до ближайшей запомненной клетки.
- **Система памяти**: FIFO-очередь с обновлением позиции при повторном посещении, чтобы забывание происходило последовательно, без разрывов.

### 10. Ресурсы

- Текстуры: PNG 32×32 пикселей (стены, пол, выход, игрок, враги, предметы, память).
- Музыка: WAV-файл, воспроизводимый через "System.Media.SoundPlayer".
- Все ресурсы встроены в проект.

### 11. Особенности реализации

- Безопасная загрузка текстур: при отсутствии ресурса используются запасные цветные фигуры, игра не падает.
- Кеширование масштабированных изображений для повышения производительности.
- Таймер игры запускается при первом движении и не останавливается до финала.
- Музыка играет непрерывно от главного меню до завершения игры.
